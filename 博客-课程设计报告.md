# 基于有状态的课堂点名系统 — Java 课程设计完整报告

## 📌 项目简介

**课堂点名系统**是一款基于 Java Swing 的桌面应用程序，旨在替代传统纸质点名方式，实现 **公平、高效、可追溯** 的课堂随机点名。系统采用 **加权随机算法**，被点名次数越少的学生被抽中概率越高，在保证随机性的同时兼顾公平性；同时内置 **救场机制**，当连续多人无法答出时自动切换策略，确保课堂节奏流畅。

- **项目名称**：基于有状态的课堂点名系统（Classroom Roll-Call System）
- **版本**：v1.0
- **开发周期**：2026 年 6 月
- **Git 仓库**：[https://github.com/Z-GOLD/roll-call-system](https://github.com/Z-GOLD/roll-call-system)

## ✨ 项目亮点

### 1. 加权随机算法（策略模式）

```java
// 核心权重计算：100 / (被点名次数 + 1)
// 被点名次数越少 → 权重越高 → 被选中概率越大
public double getCallWeight() {
    return 100.0 / (totalCalled + 1);
}
```

采用 **策略模式** 设计，`WeightedRandom` 类作为独立算法策略，可随时替换为均匀随机或其他抽取策略，符合开闭原则。

### 2. 救场机制

当课堂连续 3 名同学无法答出问题时，系统自动激活救场模式：
- 优先选择答出率高、被点名次数少的学生
- 救场持续 3 轮后自动退出
- 有效避免"冷场"尴尬，保障课堂节奏

### 3. MVC 架构

严格遵循 Model-View-Controller 分层：
- **Model**：`Student`、`RollCallRecord` — 纯数据 + 权重计算
- **View**：`MainFrame`、`StudentPanel`、`RollCallPanel`、`StatisticsPanel` — Swing 界面
- **Controller**：`RollCallService`、`StatisticsService`、`StudentDAO` — 业务逻辑与数据访问

### 4. DAO 模式数据持久化

`StudentDAO` 面向接口设计，通过对象流实现数据文件读写，未来可切换为数据库存储而无需修改上层代码。

### 5. 纯标准库实现

除 Excel 导入导出使用 Apache POI 外，项目核心功能全部基于 Java 标准库开发，无额外依赖，部署简便。
