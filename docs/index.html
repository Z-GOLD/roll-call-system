基于有状态的课堂点名系统

面向对象程序设计(Java) 课程设计 · 个人项目博客

一、项目简介

基于有状态的课堂点名系统是一个面向高校课堂教学的桌面辅助工具，旨在解决传统随机点名中"有的同学被点名多次、有的同学从未被点到"的不公平问题。

系统通过记录每位学生的历史被点名次数和回答问题次数，采用加权随机算法实现公平抽取——被点名次数越少的学生被选中的概率越大，确保在一个学期的教学过程中每位学生获得均等的被点名机会。

同时，系统内置双重救场机制：当连续 3 名同学无法回答问题时，自动从历史回答率较高的同学中选取；若救场模式也连续失败 3 次，则弹出"题目太难，建议老师讲解"提示，有效避免课堂僵局。

四、项目亮点

  1. 加权随机算法（策略模式）：权重 = 100 / (被点名次数 + 1)，采用累积权重法选择，被点名次数越少权重越高，确保点名机会公平均等
  2. 双重智能救场机制：连续 3 人未答出 → 从回答率最高的学生中选取（回答率 ≥ 最高回答率的 80%）→ 救场 3 连败 → 提示"题目太难，建议老师讲解"
  3. 完善的请假管理：支持搜索标记请假学生，点名时自动排除请假者，请假学生不计入点名统计
  4. MVC + DAO + 策略模式：三层架构分离，面向接口编程，更换存储方式（如切换为数据库）无需修改 Service 和 View 层代码
  5. 纯标准库为主：核心功能仅依赖 Java 标准库（Swing + 对象流），仅在 Excel 导入导出功能上引入 Apache POI，部署简便
